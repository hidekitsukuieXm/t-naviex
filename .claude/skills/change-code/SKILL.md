# スキル名：コード変更

## When to Use

- バグの修正や、機能を変更するとき
- コード追加・変更によって品質が低下することを防ぎたいとき

## Quick Start

1. 変更の影響範囲を調査して、指定されたissue を実装する use context7
2. ブランチを切って実装作業を行う
3. src/types/ に型定義を追加する
4. src/app/api/ にAPIルートを作成する
5. 実装したコードに対して、ESLintと単体テストおよび影響範囲のテストを実施する
6. テストが完了したら最後にプルリクエストをする
7. PRの内容をレビューする
8. プルリクエストのコメントとしてレビュー内容を記述する
9. レビュー内容の記述まで完了したら、マージしてissueをクローズする
10. 全てが完了したら、readme.mdファイルに実施した内容を追記する
