# テスト管理ツール（T-NaviEx）テスト仕様書

> 文書番号: TEST-TM-2026-001 / バージョン: 1.0 / 作成日: 2026年5月15日

---

## 1. 概要

### 1.1 目的

本テスト仕様書は、テスト管理ツール（T-NaviEx）の品質を保証するためのテスト計画・テストケースを定義します。

### 1.2 テスト対象

- フロントエンド: Next.js アプリケーション
- バックエンド: Node.js API
- データベース: PostgreSQL
- AI機能: Claude API 統合

### 1.3 テストレベル

| レベル | 説明 | ツール |
|--------|------|--------|
| 単体テスト | 個々の関数・コンポーネント | Vitest, React Testing Library |
| 結合テスト | API・DB連携 | Vitest, Supertest |
| E2Eテスト | ユーザーシナリオ | Playwright |
| 性能テスト | 負荷・応答時間 | k6, Artillery |

### 1.4 テスト環境

| 環境 | 用途 | URL |
|------|------|-----|
| ローカル | 開発者テスト | http://localhost:3000 |
| CI | 自動テスト | GitHub Actions |
| ステージング | 結合テスト | https://staging.t-naviex.example.com |
| 本番相当 | E2E・性能テスト | https://test.t-naviex.example.com |

---

## 2. テスト方針

### 2.1 テストカバレッジ目標

| 種別 | 目標 |
|------|------|
| 単体テスト（ステートメント） | 70%以上 |
| 単体テスト（ブランチ） | 60%以上 |
| API結合テスト | 主要エンドポイント100% |
| E2Eテスト | 主要ユーザーフロー100% |

### 2.2 テスト優先度

| 優先度 | 説明 | テスト頻度 |
|--------|------|-----------|
| P1 | クリティカル機能（認証、データ整合性） | 全コミット |
| P2 | 主要機能（CRUD操作、検索） | 全コミット |
| P3 | 補助機能（UI表示、フィルタ） | PR時 |
| P4 | エンハンスメント（AI機能、連携） | リリース前 |

## 11. CI/CD パイプライン設定

```yaml
# .github/workflows/test.yml

name: Test

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  unit-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run test:unit
      - run: npm run test:coverage

  integration-test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
          POSTGRES_DB: t_naviex_test
        ports:
          - 5432:5432
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx prisma migrate deploy
      - run: npm run test:integration

  e2e-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx playwright install --with-deps
      - run: npm run build
      - run: npm run test:e2e
      - uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-report
          path: playwright-report/
```
